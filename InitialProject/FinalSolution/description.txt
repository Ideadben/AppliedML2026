Clustering
    The one large outlier in columns Energy and Lz were normalized to 0 for all runs, because i reasoned it must have been a faulty measurement, since it was many standard deviations away. 
    Feature selection was done by fixing J and E as selected features (J being selected cause it was 1:1 with H and K and E being chosen for its multimodal distribution)
    and then running an exhaustive grid search along all possible feature combination values(Excluding of course K and H). In the grid search each feature combination was 
    run through a quick clustering algorithm for all possible cluster sizes from 4 to 40 (67,340 total models) with sklearn's Kmeans algorithm and n_init = 20 and optimal 
    feature combinations were determined using silhouette scores. These selected features were all used for the below models:
    ['J', 'E', 'O_FE', 'CA_FE', 'CR_FE', 'CO_FE']

    1. Clustering_BenjaminSiddique_Kmeans.csv
    Algorithm: sklearn.cluster KMeans
    Key HPs: k = 4, n_init = 50
    HP optimization: Chosen based on iterating over every cluster size and maximizing silhouette scores
    Parameters in model: 24
    Pre-processing: Scaled the input features using StandardScaler
    Loss function and value on validation set: N/A
    Own Evaluation: Looked fine in PCA decomposition but there seemed to be an isolated block that was absorbed into the really large cluster in the UMAP
    and tSNE projections which was a bit unsatisfying.
